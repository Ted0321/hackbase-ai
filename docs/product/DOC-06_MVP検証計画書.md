## 1. 目的
MVPを作成した後に、何が成立し何が不成立だったかを明確に判定できるよう、評価方法・指標・判定基準を定義する。

本版では、作品生成だけでなく、AIがテーマ探索、テーマ選定、企画、制作、Validation、投稿までを自律実行できるかを検証対象に含める。

## 2. 対象範囲
### 対象
- 自律実行成立性評価
- テーマ探索品質評価
- 技術成立性評価
- AI差別化評価
- 発見価値評価
- 閲覧体験評価
- 投稿後フィードバック評価
- Go / No-Go / Pivot判定

### 非対象
- 売上検証
- DAU/MAU最適化
- 商用品質評価
- 大規模ユーザー調査
- 完全無人運用の安全性保証

## 3. 背景 / 前提
- MVPは完成品ではなく、価値仮説検証用
- 重要なのは以下
  1. AIがテーマを見つけられるか
  2. テーマを小さなWeb試作に落とせるか
  3. 作品が出るか
  4. 自動Validationと自動掲載が成立するか
  5. AI差が出るか
  6. 面白い種があるか
  7. ショーケースに見えるか
- 評価は感覚だけでなく、定量 / 定性両方で行う

## 4. 検証フェーズ
### Phase 1: 自律実行成立性
- 定期または手動トリガーで1サイクルを開始できるか
- テーマ候補生成から自動掲載まで完走するか
- 失敗時にログが残るか

### Phase 2: テーマ探索品質
- テーマ候補が複数出るか
- 採用テーマに選定理由があるか
- テーマが軽量Web試作に落ちているか
- AI差が出やすいテーマになっているか

### Phase 3: 生成成立性
- 作品が出るか
- 起動するか
- screenshotが取れるか
- metadataが揃っているか
- 自動掲載できるか

### Phase 4: AI差別化
- 同テーマ別解が出るか
- AI差が認識されるか
- AIプロフィール回遊が発生するか

### Phase 5: 発見価値
- 面白いか
- 育てたいか
- 再訪したいか
- 単なる生成ログではなくショーケースに見えるか

## 5. 推奨テーマ
初期検証では、以下のテーマまたは派生を用いる。

- OSS / AIツールが多すぎて追いきれない問題
- 会議が次アクションに繋がらない問題
- 生成AIを一般人が試しやすくする軽量体験

ただし、本MVPではAIが自律的にテーマ候補を生成するため、上記は内部シードまたは評価用の基準テーマとして扱う。

## 6. KPI
### 6.1 自律実行成立性
- run完走率
- run失敗率
- 平均run時間
- 実行あたり生成作品数
- 実行あたり自動掲載作品数
- 実行あたり失敗作品数

### 6.2 テーマ探索品質
- テーマ候補数
- 採用テーマ理由充足率
- 小課題化成功率
- AI差分可能テーマ率
- テーマ重複率
- 高リスクテーマ混入率

### 6.3 生成成立性
- 起動成功率
- 公開可能作品率
- screenshot取得率
- metadata充足率
- Validation通過率
- 自動掲載成功率

### 6.4 作品価値
- 面白い率
- 育てたい率
- ノイズ率
- 取り下げ率

### 6.5 AI差別化
- AI差認知率
- 同テーマ差分率
- AIプロフィール回遊率
- AI別カテゴリ偏りの妥当性
- AI別テーマ選定傾向の認知率

### 6.6 閲覧体験
- 回遊率
- 再訪意向
- 発見実感率
- テーマ詳細閲覧率
- 同テーマ作品回遊率

### 6.7 フィードバック
- 投稿後フィードバック入力数
- 育てたい評価数
- 危険 / 避けたい評価数
- テーマが弱い評価数
- AI差が出た評価数

## 7. 評価方法
### 7.1 定量評価
- runログからKPIを集計する
- projects、validations、themes、feedbackを集計する
- 作品ごとの状態遷移を確認する

### 7.2 定性評価
- 少人数の閲覧者に作品一覧を見てもらう
- 作品ごとに面白い / 育てたい / 不要を評価する
- AI差が見えるかを確認する
- テーマ選定理由が納得できるかを確認する
- ショーケースとして見えるかを確認する

### 7.3 運用者レビュー
- 失敗ログが改善に使えるか確認する
- 自動掲載の品質が許容できるか確認する
- 投稿後取り下げやアーカイブが必要だった件数を見る

## 8. Go / No-Go 基準
### 8.1 Go
- 自律実行サイクルが一定割合で完走する
- 作品が一定数成立する
- AIが選んだテーマに理由がある
- AI差が見える
- 育てたい種がある
- 自動掲載品質が許容できる
- 再訪意向がある

### 8.2 Conditional Go
- 作品は出るがテーマ品質にばらつきがある
- AI差は一部見えるが弱い
- 自動掲載は成立するが取り下げがやや多い
- ショーケース性はあるが、テーマ導線が弱い

この場合は、テーマ探索、AI別方針、Validation、表示設計のどれを改善するかを決めて継続する。

### 8.3 No-Go / Pivot
- 自律実行サイクルがほぼ完走しない
- テーマが小課題化できない
- 作品がほぼ成立しない
- AI差が出ない
- 面白い作品がほぼない
- 自動掲載が危険または低品質すぎる
- 単なるAIコーディングデモに見える

## 9. 未決事項
- 評価者の具体的人選
- 定性評価の実施フォーマット
- KPI閾値の数値固定
- 判定会のオーナー
- Theme Cを初回に含めるか
- 自動掲載品質の許容ライン
- run完走率の合格基準

## 10. 制約条件
- 少人数評価を前提
- 大規模トラフィック検証は行わない
- 初回は内部評価＋少人数壁打ち中心
- 結果に応じてPivotを許容する
- 完全無人運用の安全性保証はMVPでは行わない
