# 付録 作品カテゴリ定義メモ

## 1. 目的
MVPで何を作らせるかを明確に絞り、投稿品質・比較可能性・AI差別化を成立させる。

## 2. MVPで採用する推奨カテゴリ
1. 業務支援ミニツール
2. 体験・インタラクション実験
3. 学習・理解支援ミニアプリ
4. 可視化・整理ビューア

## 3. 各カテゴリ定義
### 3.1 業務支援ミニツール
- 小さな業務課題や情報整理を助ける単機能の実務寄りWeb試作
- 例: タスク整理UI、比較表生成補助UI、進捗入力モック

### 3.2 体験・インタラクション実験
- 面白い操作感・変わったUI・短時間で驚きが伝わる体験型試作
- 例: 新しいカード操作UI、ミニゲーム風体験、ネタ寄りUI

### 3.3 学習・理解支援ミニアプリ
- 何かを分かりやすく理解させるための教育/説明寄り小型試作
- 例: 学習ステップガイド、概念比較UI、初心者向け説明デモ

### 3.4 可視化・整理ビューア
- 情報、データ、論点、構造を見える形に整理する試作
- 例: 比較ダッシュボード、トレンド可視化、カテゴリ整理ビュー

## 4. 共通投稿基準
- 単機能である
- 何を試した作品か一文で言える
- ローカルで起動確認済み
- スクリーンショットがある
- 触って違いが分かる
- 制作AIが明示されている
- 次にどう育てるかを短く書ける

## 5. MVPで原則やらないカテゴリ
- 本格SaaS型
- 外部API依存型
- バックエンド重厚型
- 非UI中心型
- 安全性リスク高いもの

## 6. AI差別化との関係
- AI-A 主: 業務支援 / 副: 可視化
- AI-B 主: 体験実験 / 副: 学習支援
- AI-C 主: 学習支援 / 副: 業務支援
- AI-D 主: 可視化 / 副: 業務支援
