# 付録 同一テーマに対するAI別出力サンプル

## 1. 目的
同一テーマに対して、AI-A〜D が本当に別方向の作品を出せるかを確認するための参考サンプル。

## 2. サンプルテーマA
### テーマ
最近話題のOSSやAIツールが増えすぎて追いきれない、という状況に対して小さなWeb試作を1つ作る。

### AI-A 業務効率化型
- タイトル案: Trend Triage Board
- 一言説明: 話題のOSSを「今すぐ試す / 監視 / 保留 / 無視」に振り分ける社内向け整理UI
- 方向性: 情報収集ではなく意思決定補助

### AI-B 遊び・実験型
- タイトル案: OSS Discovery Roulette
- 一言説明: 最近のOSSをルーレットやカードめくり形式でランダム探索する体験型UI
- 方向性: 偶然の出会いを楽しむ探索体験

### AI-C 教育・理解支援型
- タイトル案: Why This Tool Matters?
- 一言説明: 話題OSSについて、何を解決するのか・誰向けか・何が新しいのかを初学者向けに整理する学習UI
- 方向性: 難しい話題を理解可能に変換

### AI-D 可視化・分析型
- タイトル案: OSS Trend Map
- 一言説明: 最近話題のOSSをカテゴリ・用途・人気度で可視化する俯瞰UI
- 方向性: 個別具体ではなく全体構造の把握

## 3. サンプルテーマB
### テーマ
会議が多いのに、会議から実際の次アクションに繋がらない問題に対して小さなWeb試作を作る。

### AI-A
- Action Extractor Board
- 会議メモから「誰が・いつまでに・何をやるか」だけを整理する実務向け画面

### AI-B
- Meeting Chaos to Quest
- 会議で出た曖昧な話題をクエスト形式で可視化する体験型UI

### AI-C
- Why No Action? Meeting Coach
- 会議からアクションが出ない理由を分かりやすく見せる学習支援UI

### AI-D
- Meeting Flow Analyzer
- 会議内容を、論点→決定→アクションの流れで可視化する整理ビューア

## 4. 示唆
- 同一テーマでも、実務・体験・理解・可視化の4方向に分岐可能
- AI差別化は、別テーマを与えるより「同一テーマへの別解釈」で見せる方が強い
